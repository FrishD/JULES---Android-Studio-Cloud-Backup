JULES - Android Studio Cloud Backup

JULES (JetBrains User-Level Encrypted Storage) is a professional IDE plugin for Android Studio and IntelliJ IDEA. It provides automated, AES-256 encrypted backups of project files directly to the user's personal Google Drive.

Privacy Policy & Data Disclosure

Last Updated: March 4, 2026

1. Scope Justification (Why we need these permissions)

To provide secure backup services, JULES requires access to:

https://www.googleapis.com/auth/drive.file
Justification: Required to upload, update, and manage the encrypted ZIP archives created by the plugin. JULES only interacts with files it has created.

https://www.googleapis.com/auth/drive.appdata
Justification: Required to store user-specific plugin settings (e.g., backup frequency) in a hidden configuration folder, ensuring a seamless experience across multiple workstations.

2. Data Collection & Disclosure

JULES is a client-side only application. We do not operate any backend servers. We do not collect, store, transmit, or share any personal data, authentication tokens, or source code with any third parties, including the developer.

3. Google Limited Use Policy

JULES's use and transfer to any other app of information received from Google APIs will adhere to the Google API Services User Data Policy, including the Limited Use requirements.

4. Data Sharing & Security

  • No data is used for advertising, analytics, profiling, or AI training purposes.
  • All project data is encrypted locally (AES-256) before transmission.
  • OAuth access tokens are stored locally within the IDE's secure storage and never transmitted to the developer.
  • JULES does not share, sell, or rent any user data with third parties.

5. Data Retention & Deletion

Users have full control over their data. Backups can be deleted via the plugin or Google Drive. To revoke access: Google Permissions Manager.

6. Data Minimization & Scope Limitation

JULES only requests the minimal scopes required to provide its functionality. No additional data or Drive files outside the plugin's backups are accessed.

7. Transparency & User Consent

The plugin clearly informs users about the scopes requested, and users must provide explicit consent via the OAuth screen before any data access occurs.
